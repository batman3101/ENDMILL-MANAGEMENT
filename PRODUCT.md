## Users

이 앱의 사용자는 **한국·베트남 공장 작업자, 관리자, 시스템 관리자** 3계층입니다.

**1차 사용자 — 현장 작업자 (`user` role) — 스마트폰 우선**
- 자기 스마트폰(BYOD, iOS/Android 혼재)을 라인 사이를 이동하면서 사용
- 장비 옆 거치형 태블릿은 보조 (대량 입력·QR 스캔 시 사용)
- 환경: 형광등·창문 자연광 혼재, 기계 소음, 장갑 착용, 교대제 24시간 운영, 한 손 조작이 기본
- 주요 작업: 공구 교체 시 30초 안에 기록, 자기 교대 이력 조회

**2차 사용자 — 관리자 (`admin` role)**
- 사무실 데스크톱 PC 주, 현장 이동 시 자기 스마트폰 보조
- 주요 작업: 장비/엔드밀/재고 CRUD, Excel 대량 업로드, 사용자 권한 부여

**3차 사용자 — 시스템 관리자 / 경영진 (`system_admin` role)**
- 사무실 PC 위주. 보고서 출력·검토 중심
- 주요 작업: 월간/비용/공구수명/성능 보고서 검토, 공급업체 의사결정

**Job to be done**: 800대 CNC 장비의 엔드밀 공구 라이프사이클(교체 → 재고 → 발주 → 비용)을 한 시스템에서 흐름 끊김 없이 다루기.

**장면 묘사 (Scene)**: 라인 사이를 이동하는 작업자가 한 손에 자기 폰, 다른 손엔 공구를 든 채로 — 30초 안에 교체 기록을 끝낸다. 거치형 태블릿은 QR 스캐닝이나 다량 입력 같은 무거운 작업의 보조다. 사무실에서는 관리자가 27인치 모니터로 보고서를 검토하지만, 그 보고서를 만든 데이터의 90%는 그 작은 스마트폰 화면에서 입력된 것이다. 이 사실이 모든 디자인을 강제한다.

## Product Purpose

종이 기반 공구교체 로그를 디지털화하고, 800대 CNC 장비 전반의 공구 상태·재고를 실시간으로 가시화하는 **공장 운영의 신경계**입니다. 다음을 가능하게 합니다:

- 공구 교체 기록을 30초 안에 (종이 시스템 분 단위 → 30초)
- 재고 임박 자동 알림으로 stockout 0건
- 데이터 기반 의사결정: 어떤 엔드밀이 더 오래 가는가, 어느 공급업체가 비용 효율이 높은가
- 경영진이 실제로 보는 보고서 — "공장 SaaS 같다"가 아니라 "정확하고 결정 가능한"

**성공의 정의**: 작업자가 더 빨리 기록하고, 관리자가 stockout을 겪지 않으며, 경영진이 보고서를 신뢰하는 상태.

## Brand Personality

**정밀 · 장인적 · 무뚝뚝**

마이크로미터·교정기·공작기계 매뉴얼과 같은 톤. 화려하지 않고 정확하다. 숫자는 반올림 없이 정확히, 단어는 빈말 없이 사실만. SaaS의 "친근한 동반자" 톤은 거부한다 — 이것은 도구이지 친구가 아니다.

**Voice**
- 직설적, 사실 중심, 마케팅 수사 제거 ("약 1,200건"이 아니라 "1,234건")
- 정중하지만 차분함. 환영 메시지·이모지·격려 문구 거의 없음
- 작업자에게는 명령형 동사 ("기록", "확인"), 경영진에게는 데이터 자체

**감정적 목표**
- 신뢰 (데이터·보고서·알림이 정확하다)
- 효율 (모든 탭은 시간을 절약한다)
- 경의 (작업자의 시간과 경영진의 시간을 동등하게 존중)

**아닌 것**
- 친근한 챗봇 톤이 아니다
- "Delightful in a SaaS way"가 아니다
- 데이터 시각화 쇼케이스가 아니다

## Anti-references

이 앱이 절대로 닮으면 안 되는 것:

1. **AI가 생성한 SaaS 템플릿 대시보드** — 4-카드 히어로 메트릭, 그라데이션 액센트, 샘플 라인차트, "Welcome back, [Name]!" 인사. **가장 강하게 회피해야 할 패턴.**
2. **Bootstrap admin 템플릿** (AdminLTE, Tabler, ArgonAdmin) — 고채도 primary, 아이콘+제목+숫자 카드의 무한 반복.
3. **Material Design 3 반사 복붙** — 그림자 종속 카드, FAB 버튼, 기본 Roboto. 공장 운영과 어울리지 않는다.
4. **고전적 ERP UI** (SAP, Oracle EBS 스타일) — 1990년대 Windows 트리 메뉴, 겹친 창. 반대 극단도 회피.

## Design Principles

1. **데이터가 영웅이다** — 교체 로그·재고 카운트·상태 알람이 주인공. 이를 가리는 장식은 실패. 모든 디자인 결정은 "데이터가 더 명료해지는가?"로 검증.

2. **계측기 신뢰감** — 인터페이스 자체가 측정도구처럼 느껴져야 한다. 마이크로미터·교정 게이지·공작기계 매뉴얼이 참조점이다. 일관성·정확성·절제로 신뢰 축적. 팬시한 인터랙션이 신뢰를 깎으면 즉시 제거.

3. **스마트폰을 먼저, 데스크톱을 나중에** — 모든 페이지는 360px 폭 한 손 조작에서 먼저 작동해야 한다. 1차 액션은 엄지 영역(하단 60%)에, 핵심 정보는 첫 스크롤 안에. 한국어/베트남어 줄바꿈은 디자인 결정으로 다룬다 — 어절 중간이 끊어지면 디자인 실패다. 데스크톱은 모바일 레이아웃을 그리드로 펼친 결과지, 그 반대가 아니다. 표는 모바일에서 카드 리스트로 변신한다.

4. **장갑 낀 손도 통과한다** — 터치 타깃 ≥ 48px, 1차 액션 56–64px. 1m 거리에서 못 읽거나 안전구 착용 상태로 탭하지 못하면 재설계.

5. **반복은 미니멀하게, 핵심은 굵게** — 800대 장비를 스캔할 때 시각 피로를 줄인다. 상태 차이(정상/경고/위험)는 강하게, 그 외는 조용하게. 동일한 카드의 무한 반복 격자는 시각 피로의 원흉이다.

6. **카테고리 반사를 끊는다** — "CNC 공장 앱 = 진청·회색 SaaS"는 학습된 반사. 타이포·밀도·모션으로 의도적 차별화. 단, 변화를 위한 변화가 아니라 신뢰를 강화하는 차별화여야 한다.

## Accessibility & Inclusion

### 기본 a11y 규칙
- **WCAG AA 최소** (텍스트 4.5:1, 큰 텍스트 3:1) — 핵심 상태 텍스트는 AAA 지향
- **터치 타깃 ≥ 48px** (Material 표준, iOS 44px보다 +4px 여유), 1차 액션 56–64px
- **한 손 엄지 도달 가능 (Thumb Zone)** — 1차 액션은 화면 하단 영역, 보조 액션·내비는 하단 고정. 우상단의 1차 액션은 모바일에서 금지
- **가로 스크롤 표 금지** — 모바일에서 모든 표는 카드 리스트, 칩 그룹, 아코디언 등으로 변환
- **입력 폼은 한 줄 한 필드** — 모바일에서 좌우 분할 폼 금지. 키보드가 올라와도 입력 필드가 가려지지 않게 스크롤 보장
- **색약 안전** — 상태색은 형태·아이콘·라벨로 이중 인코딩 (한국 남성 적록색약 비율 5–8% 고려)
- **prefers-reduced-motion 존중** — 모션은 기능적, 장식적이지 않다
- **광원 내성** — 형광등·창문 자연광·반사를 견디는 고대비 디폴트

### 타이포그래피 & 줄바꿈 (한국어·베트남어 우선)
- **전역 `word-break: keep-all`** — CSS 디폴트(라틴어 가정)는 어절 중간을 자른다. 한국어는 공백 기준 어절 단위 끊음을 디폴트로 강제. 모든 텍스트 노드의 시작점.
- **`overflow-wrap: anywhere`는 비상시에만** — 컨테이너가 정말 좁아 어절도 못 들어갈 때 적용. URL·코드·식별자(C001, T-24) 같은 토큰 전용.
- **본문 ≥ 16px, 보조 ≥ 14px** — 자동 줌 방지 + 가독성. 작아지지 않는다.
- **`line-height` ≥ 1.6 (한국어), 1.7 (베트남어 다이아크리틱 보호)** — Latin 디폴트 1.5는 한국어/베트남어에 부족.
- **컨테이너 최소 콘텐츠 폭 ≥ 200px** — 카드·셀·버튼의 콘텐츠 영역(패딩 제외)이 200px 아래로 떨어지면 한국어 어절 줄바꿈이 무너진다. 이보다 좁아질 페이지는 콘텐츠를 줄이거나 레이아웃을 바꾼다.
- **버튼 텍스트는 1줄 강제** — 줄바꿈되는 버튼 라벨 금지. 짧은 동사("기록", "확인", "저장"), 길어지면 아이콘으로 대체.
- **헤딩 `text-wrap: balance`** — 2줄 이상 헤딩의 줄간 글자 수 자동 균형. 한쪽 줄이 짧게 끊기는 어색함 방지.
- **숫자는 tabular** — `font-variant-numeric: tabular-nums`로 모든 수치(재고, 비용, 카운트) 자릿수 정렬. 표·카드에서 시각적 안정감.
- **혼합 토큰 보호** — "PA1 모델", "T-24 위치", "C001 장비" 같은 한글+영숫자 합성은 `<span style="white-space: nowrap">`으로 감싸 한 토큰 보호. 컴포넌트 레벨 패턴화 필요.
- **다국어 타이포** — 한국어(Pretendard 권장)와 베트남어(라틴 + diacritics: đ á ề ô) 모두 지원. **BYOD 환경이라 시스템 폰트 의존 불가** — Pretendard self-host + 견고한 system fallback 필수.

### BYOD (Bring Your Own Device) 고려사항
- **iOS Safari + Android Chrome 둘 다 동등 지원** — 폰 종류·OS 버전 다양함을 가정. iOS-only 또는 Android-only 패턴 금지.
- **데이터 절약 우선** — 작업자 개인 데이터 요금제. 번들 크기 최소화, 이미지 lazy-load, 서비스 워커 캐시 적극 활용.
- **PWA "soft install"** — 강제 설치 프롬프트 금지. 브라우저에서 정상 동작이 디폴트, PWA 설치는 선택지로만 노출.
- **권한 최소화** — 카메라(QR 스캔), 위치, 알림은 사용자가 명시적으로 활성화할 때만 요청. 디폴트 OFF.
- **개인정보 분리** — 작업자 폰에 회사 데이터를 캐시하지만, 로그아웃·세션 만료 시 흔적 제거. 민감 데이터는 IndexedDB 영속 캐시가 아닌 메모리 캐시로 한정.
- **세션 만료** — 짧은 세션 + refresh token 패턴. 분실폰 위험 대응.
